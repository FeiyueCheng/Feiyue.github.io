### 表单提交
定义：将用户在input中输入的内容提交给服务器。
* 1.传统的表单提交
    * 将所有的input包裹在一个form中
    * form设置action(服务器的地址)

> 弊端：一是会进行页面的跳转。意味着服务器必须提前把一个页面写好(服务端渲染)，再将它返回给前端，前端再直接展示这个页面。二是不方便进行表单数据的渲染。   
* 2.前后端分离
    * 通过js获取到所有表单的内容。
    * 通过正则表达式进行表单的验证。
    * 发送ajax请求，将数据传递给服务器。
    * 验证成功后，服务器返回结果，需要前端解析这个数据，并且决定显示什么内容。(前端渲染和前端路由)

method:get\post
提交表单时，浏览器发送的是http请求，有两种请求方式
* 1.get 在请求URL后以？形式跟发送给服务器的参数，多个参数用&隔开。
> 由于浏览器和服务器对URL长度有限制，因此URL参数有限制，通常不能超过**1kb**
* 2.post 发给服务器的参数全部放在请求体上。

### 定位
* 标准流
    * 在标准流中，可以用margin\padding对元素定位，其中margin还可以设置负数
    * 缺点 当对一个元素进行定位，会影响其他元素的定位，不便于实现元素的层叠效果
#### CSS属性-position
* static
    * position 属性的默认值
    * 元素按照标准流布局
    * left\right\top\bottom不起作用
* relative
    * 元素按照标准流布局
    * 可以通过left\right\top\bottom进行定位
        * 定位参照对象是元素自己原来的位置
    * 应用场景
        * 在不影响其他元素位置的前提下，对当前元素位置进行微调
* fixed
    * 元素脱离标准流
    * 可以通过left\right\top\bottom进行定位
        * 定位参照对象是视口
    * 当画布滚动时，固定不动
* absolute
    * 元素脱离标准流
    * 可以通过left\right\top\bottom进行定位
        * 定位参照对象是最邻近的定位祖先元素
        * 若找不到这样的祖先元素，参照对象是视口
        * 定位祖先元素
            * position不是static的元素
    * 子绝父相(在绝大多数情况下，子元素的绝对定位都是相对与父元素进行定位)
    * 若希望子元素相对于父元素进行定位，又不希望父元素脱标，常用解决方案：父元素relative 子元素absolute

* 脱标元素的特点
* 脱标元素有 position:fixed/absolute 浮动
    * 1.可以随意设置宽高
    * 2.默认宽高是由内容决定
    * 3.不再受标准流的约束
    * 4.不再给父元素汇报宽高数据

* 绝对定位技巧
    * 绝对定位元素(absolute/fixed)
    * 对于绝对定位元素，定位参照对象的宽度 = left + right + margin-left +margin-right + 绝对定位元素的实际占用宽度(高度也是如此)

### 浮动
* 层叠关系
    * 标准元素：标准流中的元素是不存在层叠
    * 定位元素：定位元素会层叠在标准元素的上方
        * 定位元素之间可以z-index
        * 前提：必须是定位元素-非static
    * 浮动元素
    * 标准元素->浮动元素->定位元素
    
* 在css中，有三种方案对元素进行定位、布局
    * normal flow
    * absolute positioning
    * float
* 绝对定位、浮动都会脱离标准流
* float三种常用取值
    * none
    * left
    * right
* 浮动规则
    * 1.元素一旦浮动后
        * 脱离标准流
        * 朝着向左或向右移动，直到自己的边界紧贴着包含块(一般是父元素)或者其他浮动元素边界为止
    * 定位元素会层叠在浮动元素上方
    * 2.浮动元素不能与行内级内容层叠，行内级内容会被浮动元素推出
        * 比如行内级元素，inline-block元素，块级元素的文字内容
    * 3.行内级元素、inline-block元素浮动后，其顶部将与所在行的顶部对齐
    * 4.若元素向左右浮动时，不能超过包含块左右边界
    * 5.浮动元素之间不能重叠
        * 左浮找左浮，右浮找右浮
        * 若水平方向剩余的空间不够显示浮动元素，浮动元素将向下移动，直到有足够的空间为止
    * 6.浮动元素的顶端不能超过包含块的顶端，也不能超过之前所有浮动元素的顶端
* 浮动常用场景
    * 解决行内级元素、inline-block元素的水平间隙问题
    * 布局
*  问题：在进行布局时，同一行多个元素摆放后，设置margin-right让他们之间产生间距，那么最后一个元素设置的margin-right总是多余：
    * 解决1：每一行的最后一个元素，总是加一个class,通过类选择器去除margin-right
    * 解决2：伪类选择器
        * 兼容性问题
    * 解决3：.container固定的宽度>.wrap+负margin>很多item

* 浮动存在的问题
    * 由于浮动元素脱离标准流，变成了脱标元素，所以不再向父元素汇报高度
    * 父元素计算高度时，就不会计算浮动子元素的高度，导致了高度坍塌问题
    * 解决父元素高度坍塌问题的过程，一般叫做清浮动
    * 清浮动目的
        * 让父元素计算总高度时，把浮动子元素的高度算进去
* 清除浮动常用方法
    * clear常用取值
        * both:要求所有元素的顶部低于之前生成的所有浮动元素的底部
* 标准-垂直布局/定位-层叠/浮动-水平

    * flex-wrap 默认所有items在同一行显示，value为nowrap
        * wrap
        * wrap reverse

    * justify-content
        * 决定了 flex items 在main axis对齐方式
        * flex start /flex end
        * center
        * space-between/ space-evenly/ space-around

    * align-items
        * 决定了 flex items 在cross axis对齐方式
        * normal在items没有高度时，效果和strech一样，自动拉伸至填充flex container
        * flex start /flex end/ center/ baseline

    * align-content
        * 决定了多行 flex items 在cross axis对齐方式

* order 决定了排布顺序，值越小排在最前面
